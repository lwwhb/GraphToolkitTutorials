## 🎯 核心特性

### 两种图形模式

**数据流图形（Pull模式）**
- 按需评估
- 递归端口评估
- 适合：数据处理、材质生成

**执行流图形（Push模式）**
- 顺序执行
- 协程驱动
- 适合：行为树、对话系统、渲染管线

### 完整的设计模式
- 评估器模式
- 执行器模式
- 编辑器/运行时分离
- 变量和子图系统
- 上下文和块节点
